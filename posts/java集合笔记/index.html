List接口

  • List接口存储的数据是有序的、可重复的

ArrayList

  1. 线程不安全
  2. 可存储不同类型的对象
  3. 底层使用数组存储数据，默认情况下是长度为10。当容量不足使扩容为原来容量的1.5倍
  4. 需重写equals()方法
  5. 添加元素效率高

LinkedList

  1. 线程不安全
  2. 可存储不同类型的对象
  3. 底层使用双向链表存储数据
  4. 需重写equals()方法
  5. 插入元素、删除元素效率高

Vector

  1. 线程安全
  2. 可存储不同类型的对象
  3. 底层使用数组存储数据，默认情况下是长度为10。当容量不足使扩容为原来容量的2倍
  4. 需重写equals()方法
  5. 添加元素效率高
  6. ArrayList的古老实现类

Set接口

  • Set接口存储的数据是无序的、不可重复的

HashSet

  1. 线程不安全。
  2. 可存储不同类型的对象。
  3. 底层使用HashMap。
  4. 需重写equals()、hashCode()方法。
  5. 底层同时使用了数组和链表，存储数据时，通过计算hashcode得到应存放在在数组中的位置，如果此位置没有元素，则直接存入；有元素则判断hashcode是否相同，如若不相同则此位置元素指向新加入的元素（jdk8.0及以后）（jdk7.0及以前则放入新元素指向旧元素）；如若hashcode相同则调用新加入元素的equals()方法比较在此位置上的元素，如若不同则存入（此位置元素指向新加入的元素（jdk8.0及以后））；如若相同则存入失败。

LinkedHashSet

  1. LinkedHashSet是HashSet的子类
  2. 线程不安全。
  3. 可存储不同类型的对象。
  4. 底层使用LinkedHashMap。
  5. 需重写equals()、hashCode()方法。
  6. 底层在HashSet的基础上加入了双向链表来记录元素的加入顺序。

TreeSet

  1. 线程不安全。
  2. 不可存储不同类型的对象。
  3. 底层使用TreeMap。
  4. 加入的元素需继承Comparable接口（compareTo()方法）或使用Comparator接口（compare()方法）。
  5. 底层使用红黑树，加入元素时，小的放左子树，大的放右子树。

Map接口

  • Map接口存储

HashMap

  1. 线程不安全，效率高。
  2. 可存储key或value为null的数据。
  3. HashMap由key、value、entry构成，一对key-value构成了entry。key使用Set存储；value使用collection存储；entry使用set。
  4. key是无序的、不可重复的；value是无序的、可重复的；entry是无序的、不可重复的。
  5. HashMap使用数组和链表存储数据（jdk7.0及以前）（jdk8.0使用数组、链表和红黑数）。
  6. jdk7.0及以前底层使用entry数组存储数据，jdk8.0使用node数组。
  7. 数组在达到临界值时扩容，临界值为原来数组长度乘0.75，扩容长度为原来长度的2倍。
  8. jdk8.0之后在链表长度大于8并且数组长度大于64时，会将链表转成红黑树；在链表长度小于6时，会将红黑树转为链表。

LinkedHashMap

  1. LinkedHashMap是HashMap的子类
  2. 底层继承了HashMap的node，名字为entry，可记录元素的加入顺序。

TreeMap

  1. 线程不安全。
  2. 不可存储不同类型的对象。
  3. 加入的元素需继承Comparable接口（compareTo()方法）或使用Comparator接口（compare()方法）。
  4. 底层使用红黑树，加入元素时，小的放左子树，大的放右子树。

Hashtable

  1. 线程安全，效率低
  2. HashMap的古老实现类。
  3. 不可存储key或value为null的数据
