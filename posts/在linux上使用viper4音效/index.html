在Linux上使用VIPER4音效

2019-09-12
3分钟阅读时长
软件相关

VIPER4音效是什么？

摘自百度百科：VIPER4Android

ViPER4Android XHiFi 是一款用于Android系统的全局音效驱动，主要用于还原音频中原有的声音细节，包括回放增益、均衡器、音频重构和限幅器。并于2013年9月5日更新至2.1.0.2-1后停止更新，其功能并入FX版本。（注意：作者并未明确表示XHIFI版可支持安卓4.4） ViPER4Android FX 同样是一款用于Android系统的全局音效驱动，主要用于给音频渲染各种各样的音频效果，包括回放增益、均衡器、脉冲反馈处理、场环绕、数字混响、动态系统和限幅器。无浮点运算的机型(如采用PXA920 CPU的机器)不支持脉冲反馈处理和耳机环绕核擎+效果。持续更新至今。(注：当前版本驱动选择也受到平台限制，无NEON模块的平台无法选择高质量以及发烧级驱动)


VIPER4论坛：http://forum.vipersaudio.com/

支持平台：

  • Android
  • Windows
  • Linux (Github)

大家所熟知的有两个版本，安卓平台VIPER4Android简称V4A；Windows平台VIPER4Windows简称V4W。博主在逛Github中发现了VIPER4的Linux版本VIPER4Linux。


安装VIPER4Linux

VIPER4Linux项目地址：https://github.com/noahbliss/Viper4Linux

如果你使用ArchLinux的话输入这条命令直接安装：

 yay -S viper4linux-git

其他发行版或者需要手动安装详见项目介绍：https://github.com/noahbliss/Viper4Linux


安装图形前端

如果需要图形界面的话，则另外还需要安装一个GUI前端的包：

 yay -S viper4linux-gui-git

VIPER4Linux图形前端项目地址：https://github.com/ThePBone/Viper4Linux-GUI


配置

先在~/.config/viper4linux目录下新建一个文件：devices.conf

 touch ~/.config/viper4linux/devices.conf

如果你的shell环境是英文,则命令行输入：

 pactl list sinks | grep "Name: " -A1

如果你的shell环境是中文,则命令行输入：

 pactl list sinks | grep "名称：" -A1

找到 Name: 或者 名称： 后面的字段，复制

打开devices.conf 写入下面内容：

location=把Name: 或者 名称：后面字段复制进来

比如我输入 pactl list sinks | grep “名称：” -A1 结果是：

$ pactl list sinks | grep "名称：" -A1
名称：alsa_output.pci-0000_00_1f.3.analog-stereo
描述：内置音频 模拟立体声

则在devices.conf文件中写入

location=alsa_output.pci-0000_00_1f.3.analog-stereo

使用

完成上面的配置后就可以启动了

启动 VIPER4Linux

viper start

重新启动

viper restart

关闭

viper stop

查看状态

viper status

配置音效

音效的配置在 ~/.config/viper4linux/audio.conf 中 可按自己的喜好修改

建议配合VIPER4Linux图形前端进行管理配置

脉冲反馈样本 可在 ~/ 下建立一个名为IRS的文件夹 把样本放在文件夹配合 VIPER4Linux图形前端进行管理

VIPER4音效配置：https://github.com/noahbliss/Viper4Linux-Configs

更多音效配置或者脉冲反馈样本可在 VIPER4论坛：http://forum.vipersaudio.com/ 自行寻找
