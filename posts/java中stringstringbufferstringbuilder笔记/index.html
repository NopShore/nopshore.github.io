Java中String、StringBuffer、StringBuilder笔记

2022-04-19
2分钟阅读时长

String

  1. String类final修饰的，不可继承。
  2. String内部使用private final修饰的char[]数组存放字符串，private使外部无法修改char[]数组内容，final使其初始化后无法指向其他数组，所以说String类是不可变的。每次对String引用的赋值，都是重新创建一个对象把地址赋给引用。
  3. String类中对字符串修改的方法通常返回一个新的String类对象。
  4. String s1 = "hello"，以此种方式赋值，“hello"实际存放在方法区中的字符串常量池里。s1指向常量池中"hello"的地址。String s2 = new String("hello2")，此种方式创建对象，会在堆中创建一个String对象，堆中的String对象再指向常量池中的"hello2"字符串。

StringBuffer

  1. StringBuffer是线程安全的，效率比StringBuilder低。
  2. StringBuffer使用new创建对象时，可以传入int类型参数来指定内部char数组长度。
  3. StringBuffer使用new创建对象时，以传入的字符串的长度+16来指定内部char数组长度。
  4. 像apped类的操作，如果超出内部char数组长度，则会创建一个新的char数组，长度为原字符串的两倍+2，把旧数组的所有内容拷贝给新数组，然后指向新数组。

StringBuilder

  1. StringBuilder线程不安全，但是效率比StringBuffer高。
